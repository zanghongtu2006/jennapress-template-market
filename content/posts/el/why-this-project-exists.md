Πολλά website projects ξεκινούν ως απλές εργασίες δημοσίευσης, αλλά σταδιακά συσσωρεύουν runtime APIs, ad hoc κανόνες επεξεργασίας και αποφάσεις deployment που κανείς δεν κατέχει πραγματικά. Το αποτέλεσμα δεν είναι ισχύς. Το αποτέλεσμα είναι σύγχυση.

Το Jenna Press υπάρχει επειδή αυτή η διολίσθηση είναι συνηθισμένη και αποφεύξιμη. Μια μικρή ομάδα συνήθως δεν χρειάζεται ιδιωτικό server layer μόνο και μόνο για να εξηγήσει ένα έργο, να δημοσιεύσει έναν οδικό χάρτη ή να συντηρεί ένα πολυγλωσσικό blog. Χρειάζεται μια σταθερή δομή που δεν αλλάζει κάθε εβδομάδα.

Το έργο λοιπόν ξεκινά από μια διαφορετική ερώτηση: ποιο είναι το μικρότερο framework που εξακολουθεί να επιτρέπει σε μια ομάδα να παραδώσει με σιγουριά έναν σοβαρό ιστότοπο έργου;
